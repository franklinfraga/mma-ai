# Configuration Directory

This directory contains optimized parameters for the MMA AI prediction system.

## Files

### `optimized_parameters.json`

Statistically optimized smoothing parameters generated by `tuning/comprehensive_likelihood_tuner.py`.

**Structure:**
```json
{
  "metadata": {
    "training_period": "2014-01-01 to 2023-01-01",
    "n_fights": <number>,
    "optimized_at": "<timestamp>"
  },
  "beta_binomial": {
    "global": { "ko": <tau>, "win": <tau>, ... },
    "per_weightclass": {
      "featherweight": { "sub_land": <tau> },
      ...
    }
  },
  "poisson_gamma": {
    "global": { "sig_str": <tau>, "td": <tau>, ... },
    "per_weightclass": { ... }
  }
}
```

**How It Works:**

1. **Automatic Generation**: If this file doesn't exist, `main.py` will automatically run the optimization process (takes 30-60 minutes)

2. **Intelligent Caching**: The file is validated before each run:
   - Training period must match (2014-2023)
   - Fight count must be within ±100 of current database
   - Can be forced to reoptimize with `FORCE_REOPTIMIZE=1`

3. **Per-Weightclass Parameters**: Different weight classes use statistically validated tau values:
   - If a weightclass-specific parameter exists (e.g., featherweight `sub_land`), use it
   - Otherwise, fall back to global parameter
   - If neither exists, fall back to baseline hardcoded parameter

4. **Usage Modes**:
   ```bash
   # Default: Use optimized parameters (this file)
   python main.py

   # Force reoptimization (regenerate this file)
   FORCE_REOPTIMIZE=1 python main.py

   # Use baseline hardcoded parameters (ignore this file)
   PARAM_MODE=baseline python main.py
   ```

## Manual Regeneration

To manually regenerate optimized parameters:

```bash
python tuning/comprehensive_likelihood_tuner.py
```

This will:
- Run grid search optimization for beta-binomial tau values
- Run grid search optimization for poisson-gamma tau values
- Use time-series cross-validation (training: 2014-2023, test: 2023-2024)
- Save results to `config/optimized_parameters.json`

## Version Control

This file is **generated** and should generally not be committed to version control unless you want to share specific optimized parameters across team members. Add to `.gitignore` if you want each environment to generate its own parameters.
